What's Nmap?

Network and port tests, which are among the basic security elements, are very important. This test is necessary to monitor and test the vulnerabilities in the running services, any deficiency in the network and security measures. Although we think it is safe, new security vulnerabilities are always emerging. It should be done in server-related tests as well as testing web applications.

Common vulnerabilities in networks are ports and running services. If there is a vulnerability in the software used on the server, it becomes much easier for the attacker to take over the Server. Nmap will also make it significantly easier to find these vulnerabilities.

At the same time, the Nmap tool is the most stable and most used tool for port scans. You can scan ports for all types of systems and obtain different information with many scanning types in it. You can use it not only for port scans, but also for operations such as detecting operating systems, network mapping, firewall tests.

Nmap has a public license. You can access the open source code and develop it as you wish. The Nmap tool is included with Kali Linux.

Nmap Scan Types

There are many types of scans in the nmap tool. You can perform different types of scans according to the system and according to the services running on the system, firewalls. By typing nmap into the terminal, you can see the types of scans in a sequential manner.

Host Discovery

It is important to explore hosts on a network or on any server. We can simply list these hosts.

' nmap -sP -T4 target-ip '

We can perform our own scan. In a simple way, we perform a ping scan with the '-sP' parameter and speed up our scan in the form of '-T4'.

This scan sends SYN packets to the 80 ports of the hosts on the network in a simple way, giving us the output according to the responses.

We can also get our goals from a txt file.

' nmap -sV -iL /root/Desktop/hosts.txt '

We performed our first scan. We scanned the services of the targets we received from the txt file with the '-iL' parameter with the '-sV' parameter, as can be seen. This way you can perform a batch scan.

TCP SYN SCAN

It sends TCP Syn packets to the target. If the ports are open, the SYN-ACK response will return. if the machine is making a call, the scan will end by sending Dec RST packet.

We can complete our scan in the form of 'nmap -sS target-ip'.

ACK SCAN

In the meantime, Dec sends TCP ACK packets to the destination. If no response is returned, or if an ICMP Destination Unreachable response is received, the ports are returned as "filtered". So it responds to us with filters.

If the targer sends an RST-flagged packet, the ports return "unfiltered", that is, unfiltered.

We have performed our scan in the form of 'nmap -sA target-ip -p target-port'. After our '-sA' parameter, we set the destination and selected the port to be scanned with the '-p' parameter. After the scan is performed, it responds to you as "filtered" or "unfiltered".

FIN SCAN

In this scan, packets with FIN flags are sent to the target. If the closed port is detected during a batch port scan, the packet with the RST - ACK flag is returned. If the port is open, no response is returned.

The '-sF' parameter is used in this scan. In this way, you can perform the scan or you can perform the scan directly without specifying a port.

WINDOW SCAN

According to the size of the RST frame of the responses returned in the form of this scan, we can understand whether the port is open or not. If it is 0, it is closed. if it is in a size other than 0, it means that the port is open.

In this scan, we use the '-sW' parameter. As it turned out, our scan was performed. And also you can see the open ports by checking the rotating packets with the Wireshark tool.

TCP CONNECT SCAN

This is similar to the "TCP SYN Scan" in my scan. This similarity is the RST - ACK packets that the target will send in cases where the ports are closed.

In obvious cases, the scanning machine in response to the target's RST-ACK packets sends ACK-flagged packets to the target and the handshake is completed.

We complete it in the form of 'nmap -sT target-ip'. You can also check the parameters from the table I have prepared in advance.

PING SCAN

In this type of scan, a single packet is sent and the destination ICMP Echo sends the response. If the sent packet is subjected to filtering, the destination machine does not send any response. You can perform this scan using the '-sP' parameter.

Here, the scanning machine sends UDP packets to the target machine. If the destination machine responds with UDP packets, it means that the ports are open. If the ICMP Port returns an Unreachable response, it means that it is closed, and if it does not give any response, it means that it is open - filtered. The parameter used in the scan is the '-sU' parameter. This is how you can perform a scan.

IP PROTOCOL SCAN

In this type of scan, the source machine sends IP packets to the destination, and the destination machine responds to these packets with RST packets. If access is not provided, it will not return any response. The '-sO' parameter is used in this scan.
